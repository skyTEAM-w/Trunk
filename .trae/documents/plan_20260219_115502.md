- **Project Overview**: Purpose and functionality of the Truck Predictive Maintenance System.

- **Technical Stack**: Java 17, Spring Boot 3.x, MySQL 8.0, DJL (Deep Java Library), Python integration.

- **API/Controller Reference**: Mapping of URLs to Controllers.

## 2. Refactoring to Modern Architecture (Branch: `refactor/modern-architecture`)
I will create a new git branch and perform the following refactoring steps:

### A. Project Structure & Configuration
1.  **Spring Boot Conversion**:
    -   Update `pom.xml` to fully utilize Spring Boot 3 features (packaging as `jar`).
    -   Create `TruckApplication.java` as the main entry point.
    -   Remove `web.xml` and migrate configurations to Spring Boot code/properties.
2.  **Configuration Management**:
    -   Migrate `db_connect.xml` and `python_host.xml` to `src/main/resources/application.yml`.
    -   Create `@ConfigurationProperties` classes to inject these settings type-safely.
    -   **Constraint Check**: Ensure no configuration defaults to C: drive; all paths will be relative or configurable.

### B. Backend Refactoring
1.  **Data Access Layer**:
    -   Replace manual `JDBC_UTL` with Spring's `JdbcTemplate` (or Spring Data JDBC).
    -   Convert `Dao` implementations to `@Repository` classes.
    -   Inject `DataSource` automatically via Spring Boot.
2.  **Service Layer**:
    -   Annotate existing services with `@Service`.
    -   Use Constructor Injection instead of `new ServiceImpl()`.
3.  **Web Layer**:
    -   Convert `HttpServlet` classes to Spring `@Controller`.
    -   Map endpoints using `@GetMapping` and `@PostMapping`.
    -   Replace manual `RequestDispatcher` with return values (view names).

### C. Frontend Migration (JSP to Thymeleaf)
To support a modern, portable JAR deployment:
1.  **Static Resources**: Move `src/main/webapp/css`, `js`, `img` to `src/main/resources/static`.
2.  **Templates**:
    -   Convert `.jsp` files to Thymeleaf `.html` templates in `src/main/resources/templates`.
    -   Update syntax from JSP EL (`${}`) to Thymeleaf (`th:text="${}"`, `th:action="@{...}"`).
    -   This eliminates the need for a standalone Tomcat server.

### D. Python Integration
-   Refactor `HttpCommunicationLayer` to use Spring's `RestTemplate` or `WebClient` (modern approach) or keep as a managed `@Component` injected with configuration properties.

## 3. Verification
-   Compile the project to ensure no build errors.
-   Verify that the application starts up (simulated check of logs/console).
-   Ensure all "C drive" restrictions are respected in the new configuration.
